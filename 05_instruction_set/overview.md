# 5.1 指令集概述

CoralNPU 是一个基于 RISC-V 指令集架构（ISA）的机器学习加速器。本章将详细介绍 CoralNPU 支持的指令集及其编码格式。

## 什么是指令集

**指令集**（Instruction Set Architecture，ISA）是处理器能够理解和执行的所有指令的集合。你可以把它理解为处理器的"语言"——就像人类有中文、英文等不同语言一样，不同的处理器也有不同的指令集。

指令集定义了：
- 处理器能执行哪些操作（加法、减法、跳转等）
- 这些操作如何用二进制数字编码
- 处理器有哪些寄存器（临时存储数据的地方）
- 如何访问内存

RISC-V 是一种开源的指令集架构，它的设计理念是"精简指令集"（RISC = Reduced Instruction Set Computer），即用少量简单的指令组合完成复杂的任务。

## CoralNPU 支持的指令集

CoralNPU 实现了以下 RISC-V 指令集扩展：

```
rv32imf_zve32x_zicsr_zifencei_zbb
```

这个字符串看起来很复杂，让我们逐个解释：

### 基础指令集

- **RV32I**：RISC-V 32 位整数基础指令集
  - "RV" 表示 RISC-V
  - "32" 表示 32 位架构（寄存器和地址都是 32 位）
  - "I" 表示整数（Integer）指令

- **RV32M**：整数乘法和除法扩展
  - 提供乘法（mul）、除法（div）、取余（rem）等指令

- **RV32F**：单精度浮点扩展
  - 支持 32 位浮点数运算（float 类型）
  - 提供浮点加减乘除、比较等指令

### 向量扩展

- **Zve32x**：嵌入式向量扩展（32 位元素）
  - 这是 RISC-V 向量扩展（RVV）的一个子集
  - 专为嵌入式系统设计，支持整数向量运算
  - "32" 表示向量元素最大宽度为 32 位
  - "x" 表示只支持整数，不支持浮点向量

### 系统扩展

- **Zicsr**：控制和状态寄存器（CSR）指令
  - 提供读写 CSR 的指令（csrrw、csrrs、csrrc）
  - CSR 用于配置处理器行为和查询状态

- **Zifencei**：指令屏障扩展
  - 提供 fence.i 指令
  - 用于同步指令缓存和数据缓存（例如自修改代码场景）

- **Zbb**：基本位操作扩展
  - 提供位计数、位旋转、位反转等指令
  - 例如：clz（计算前导零）、ctz（计算尾随零）、rol（循环左移）等

## 指令集组成

CoralNPU 的指令集可以分为三大类：

### 1. 标量指令

标量指令一次处理一个数据，包括：

- **算术运算**：加、减、乘、除
- **逻辑运算**：与、或、异或、移位
- **位操作**：位计数、位旋转（Zbb 扩展）
- **分支跳转**：条件分支、无条件跳转
- **访存指令**：加载（load）、存储（store）
- **浮点运算**：浮点加减乘除、浮点比较（F 扩展）

### 2. 向量指令

向量指令一次处理多个数据（SIMD），包括：

- **向量算术**：向量加减乘除
- **向量访存**：向量加载、向量存储
- **向量配置**：设置向量长度和元素宽度

### 3. 系统指令

系统指令用于控制处理器行为，包括：

- **CSR 操作**：读写控制寄存器
- **同步指令**：fence、fence.i
- **特权指令**：ecall、ebreak、mret

## 指令编码格式

RISC-V 指令采用固定长度编码，每条指令都是 **32 位**（4 字节）。

> **注意**：标准 RISC-V 支持 16 位压缩指令（C 扩展），但 CoralNPU 回收了 C 扩展的编码空间，用于支持更多的 SIMD 寄存器（64 个向量寄存器需要 6 位索引）。

### RISC-V 的六种基本指令格式

RISC-V 定义了 6 种基本的指令编码格式，分别是：R、I、S、B、U、J 型。

每种格式都将 32 位指令划分为不同的字段（field），每个字段有特定的含义：

- **opcode**：操作码，标识指令类型（7 位）
- **rd**：目标寄存器（destination register，5 位）
- **rs1**：源寄存器 1（source register 1，5 位）
- **rs2**：源寄存器 2（source register 2，5 位）
- **funct3**：功能码 3（3 位），进一步区分指令
- **funct7**：功能码 7（7 位），进一步区分指令
- **imm**：立即数（immediate），指令中直接包含的常数值

下面用 ASCII 图展示这 6 种格式：

#### R 型指令（寄存器-寄存器运算）

用于两个寄存器之间的运算，结果写入第三个寄存器。

```
 31        25 24    20 19    15 14  12 11     7 6      0
+------------+--------+--------+------+--------+--------+
|   funct7   |   rs2  |   rs1  |funct3|   rd   | opcode |
+------------+--------+--------+------+--------+--------+
    7 bits     5 bits   5 bits  3 bits  5 bits   7 bits
```

**示例**：`add x3, x1, x2`（x3 = x1 + x2）

**解释**：
- `funct7` 和 `funct3` 组合起来确定具体是哪种运算（加法、减法、与、或等）
- `rs1` 和 `rs2` 是两个源操作数寄存器
- `rd` 是目标寄存器，存放运算结果

#### I 型指令（立即数运算）

用于寄存器和立即数之间的运算，或者加载指令。

```
 31                  20 19    15 14  12 11     7 6      0
+----------------------+--------+------+--------+--------+
|      imm[11:0]       |   rs1  |funct3|   rd   | opcode |
+----------------------+--------+------+--------+--------+
       12 bits           5 bits  3 bits  5 bits   7 bits
```

**示例**：`addi x3, x1, 10`（x3 = x1 + 10）

**解释**：
- `imm[11:0]` 是 12 位立即数（常数），会被符号扩展到 32 位
- `rs1` 是源寄存器
- `rd` 是目标寄存器

**符号扩展**：如果立即数的最高位是 1，表示负数，扩展时高位全填 1；如果是 0，高位全填 0。

#### S 型指令（存储指令）

用于将寄存器的值存储到内存。

```
 31        25 24    20 19    15 14  12 11     7 6      0
+------------+--------+--------+------+--------+--------+
| imm[11:5]  |   rs2  |   rs1  |funct3|imm[4:0]| opcode |
+------------+--------+--------+------+--------+--------+
    7 bits     5 bits   5 bits  3 bits  5 bits   7 bits
```

**示例**：`sw x2, 8(x1)`（将 x2 的值存储到地址 x1+8）

**解释**：
- 立即数被拆分成两部分：`imm[11:5]` 和 `imm[4:0]`
- `rs1` 是基地址寄存器
- `rs2` 是要存储的数据寄存器
- 存储地址 = rs1 + imm（符号扩展）

**为什么立即数要拆分**？因为 S 型指令没有目标寄存器 rd（存储指令不需要写回寄存器），所以 rd 的位置被用来存放立即数的低 5 位。

#### B 型指令（分支指令）

用于条件跳转。

```
 31   30        25 24    20 19    15 14  12 11    8 7    6      0
+----+------------+--------+--------+------+-------+----+--------+
|imm[12]|imm[10:5]|   rs2  |   rs1  |funct3|imm[4:1]|imm[11]| opcode |
+----+------------+--------+--------+------+-------+----+--------+
 1bit    6 bits     5 bits   5 bits  3 bits  4 bits  1bit   7 bits
```

**示例**：`beq x1, x2, label`（如果 x1 == x2，跳转到 label）

**解释**：
- 立即数被拆分成 4 部分：`imm[12]`、`imm[11]`、`imm[10:5]`、`imm[4:1]`
- 注意：没有 `imm[0]`，因为 RISC-V 指令地址必须是 2 字节对齐的，最低位永远是 0
- 跳转目标地址 = PC + imm（符号扩展）
- `rs1` 和 `rs2` 是要比较的两个寄存器

**为什么立即数这么碎**？为了在固定的 32 位中塞下两个源寄存器和一个 13 位立即数，同时保持 opcode 和 funct3 的位置一致。

#### U 型指令（高位立即数）

用于加载 20 位立即数到寄存器的高 20 位。

```
 31                                12 11     7 6      0
+------------------------------------+--------+--------+
|            imm[31:12]              |   rd   | opcode |
+------------------------------------+--------+--------+
              20 bits                  5 bits   7 bits
```

**示例**：`lui x3, 0x12345`（x3 = 0x12345000）

**解释**：
- `imm[31:12]` 是 20 位立即数，会被放到目标寄存器的高 20 位
- 低 12 位自动填 0
- 常与 `addi` 配合使用，先用 `lui` 加载高 20 位，再用 `addi` 加载低 12 位

#### J 型指令（无条件跳转）

用于无条件跳转并保存返回地址。

```
 31   30          21 20   19          12 11     7 6      0
+----+--------------+----+---------------+--------+--------+
|imm[20]|imm[10:1]|imm[11]|  imm[19:12]  |   rd   | opcode |
+----+--------------+----+---------------+--------+--------+
 1bit    10 bits    1bit     8 bits        5 bits   7 bits
```

**示例**：`jal x1, label`（跳转到 label，并将返回地址保存到 x1）

**解释**：
- 立即数被拆分成 4 部分：`imm[20]`、`imm[19:12]`、`imm[11]`、`imm[10:1]`
- 跳转目标地址 = PC + imm（符号扩展）
- 返回地址（PC+4）保存到 rd
- 用于函数调用

### 为什么有这么多种格式？

每种格式都是为了在 32 位的限制下，尽可能高效地编码不同类型的指令：

- **R 型**：需要 3 个寄存器（2 个源 + 1 个目标）
- **I 型**：需要 2 个寄存器 + 1 个立即数
- **S 型**：需要 2 个寄存器 + 1 个立即数，但不需要目标寄存器
- **B 型**：需要 2 个寄存器 + 1 个跳转偏移
- **U 型**：需要 1 个寄存器 + 1 个大立即数
- **J 型**：需要 1 个寄存器 + 1 个大跳转偏移

### 立即数为什么要拆分？

你可能注意到 B 型和 J 型指令的立即数被拆得很碎。这是因为：

1. **保持字段位置一致**：让 opcode、funct3、rs1、rs2、rd 在不同格式中尽量在相同位置，简化硬件解码逻辑
2. **节省空间**：在固定的 32 位中塞下所有必要信息
3. **对齐要求**：RISC-V 指令地址必须 2 字节对齐，所以立即数的最低位可以省略

## 指令长度

CoralNPU 的所有指令都是 **32 位定长**。

标准 RISC-V 支持 16 位压缩指令（C 扩展），可以减少代码体积。但 CoralNPU 为了支持更多的向量寄存器（64 个向量寄存器需要 6 位索引，而标准 RISC-V 只有 5 位寄存器索引），回收了 C 扩展的编码空间。

**优点**：
- 简化指令解码逻辑
- 支持更多向量寄存器
- 提高向量运算性能

**缺点**：
- 代码体积略大（相比支持 C 扩展的处理器）

## 寄存器约定

CoralNPU 有三类寄存器：

### 1. 通用寄存器（标量寄存器）

共 32 个，每个 32 位宽：

```
寄存器编号    ABI 名称    用途说明
-----------  ----------  ----------------------------------
x0           zero        硬连线为 0，写入无效
x1           ra          返回地址（return address）
x2           sp          栈指针（stack pointer）
x3           gp          全局指针（global pointer）
x4           tp          线程指针（thread pointer）
x5-x7        t0-t2       临时寄存器（temporary）
x8           s0/fp       保存寄存器 / 帧指针
x9           s1          保存寄存器
x10-x11      a0-a1       函数参数 / 返回值
x12-x17      a2-a7       函数参数
x18-x27      s2-s11      保存寄存器
x28-x31      t3-t6       临时寄存器
```

**ABI 名称**（Application Binary Interface）是软件约定的寄存器用途，硬件本身不强制。

**重要概念**：
- **临时寄存器（t0-t6）**：函数调用时不保证保留，调用者需要自己保存
- **保存寄存器（s0-s11）**：函数调用时被调用者需要保存和恢复
- **参数寄存器（a0-a7）**：传递函数参数，a0-a1 也用于返回值

### 2. 浮点寄存器

共 32 个，每个 32 位宽（单精度浮点）：

```
寄存器编号    ABI 名称    用途说明
-----------  ----------  ----------------------------------
f0-f7        ft0-ft7     浮点临时寄存器
f8-f9        fs0-fs1     浮点保存寄存器
f10-f11      fa0-fa1     浮点参数 / 返回值
f12-f17      fa2-fa7     浮点参数
f18-f27      fs2-fs11    浮点保存寄存器
f28-f31      ft8-ft11    浮点临时寄存器
```

### 3. 向量寄存器

共 64 个，每个 256 位宽：

```
寄存器编号    名称        说明
-----------  ----------  ----------------------------------
v0-v63       v0-v63      向量寄存器（256 位 = 8 个 32 位元素）
```

**向量寄存器的特点**：
- 每个寄存器可以存储 8 个 32 位整数（int32 x 8）
- 也可以存储 16 个 16 位整数（int16 x 16）
- 或者 32 个 8 位整数（int8 x 32）
- 元素宽度由向量配置指令（vsetvl）设置

### 4. 控制和状态寄存器（CSR）

CSR 用于配置处理器行为和查询状态，通过 Zicsr 扩展的指令访问。

常用的 CSR 包括：

```
地址      名称        说明
-------  ----------  ----------------------------------
0x001    fflags      浮点异常标志
0x002    frm         浮点舍入模式
0x003    fcsr        浮点控制和状态寄存器
0x009    vxsat       向量饱和标志
0x00A    vxrm        向量舍入模式
0x00F    vcsr        向量控制和状态寄存器
0xC00    cycle       周期计数器（低 32 位）
0xC01    time        时间计数器（低 32 位）
0xC02    instret     指令计数器（低 32 位）
```

## 指令分类

CoralNPU 的指令可以按功能分为以下几类：

### 1. 标量算术和逻辑指令

**整数运算**（RV32I + RV32M）：
- 加减法：`add`、`sub`、`addi`
- 比较：`slt`、`sltu`、`slti`、`sltiu`
- 逻辑运算：`and`、`or`、`xor`、`andi`、`ori`、`xori`
- 移位：`sll`、`srl`、`sra`、`slli`、`srli`、`srai`
- 乘除法：`mul`、`mulh`、`div`、`rem`

**位操作**（Zbb）：
- 位计数：`clz`（前导零）、`ctz`（尾随零）、`cpop`（1 的个数）
- 位旋转：`rol`（循环左移）、`ror`（循环右移）
- 最大最小：`max`、`min`、`maxu`、`minu`
- 符号扩展：`sextb`、`sexth`
- 零扩展：`zexth`
- 位反转：`rev8`

**浮点运算**（RV32F）：
- 算术：`fadd.s`、`fsub.s`、`fmul.s`、`fdiv.s`
- 融合乘加：`fmadd.s`、`fmsub.s`
- 比较：`feq.s`、`flt.s`、`fle.s`
- 转换：`fcvt.w.s`（浮点转整数）、`fcvt.s.w`（整数转浮点）

### 2. 访存指令

**标量访存**：
- 加载：`lb`、`lh`、`lw`、`lbu`、`lhu`（加载字节/半字/字）
- 存储：`sb`、`sh`、`sw`（存储字节/半字/字）
- 浮点加载存储：`flw`、`fsw`

**向量访存**（Zve32x）：
- 单位步长：`vle32.v`、`vse32.v`（连续加载/存储）
- 跨步访问：`vlse32.v`、`vsse32.v`（按固定步长加载/存储）
- 索引访问：`vlxei32.v`、`vsxei32.v`（按索引加载/存储）

### 3. 分支和跳转指令

**条件分支**：
- `beq`：相等则跳转
- `bne`：不等则跳转
- `blt`：小于则跳转（有符号）
- `bge`：大于等于则跳转（有符号）
- `bltu`：小于则跳转（无符号）
- `bgeu`：大于等于则跳转（无符号）

**无条件跳转**：
- `jal`：跳转并链接（用于函数调用）
- `jalr`：寄存器跳转并链接（用于函数返回和间接跳转）

### 4. 向量指令

**向量配置**：
- `vsetvl`：设置向量长度和元素宽度

**向量算术**：
- 向量加减：`vadd.vv`、`vsub.vv`
- 向量乘法：`vmul.vv`
- 向量移位：`vsll.vv`、`vsrl.vv`

**向量访存**：
- 见上文"访存指令"部分

### 5. 系统指令

**CSR 操作**（Zicsr）：
- `csrrw`：读写 CSR（原子交换）
- `csrrs`：读 CSR 并置位
- `csrrc`：读 CSR 并清位

**同步指令**：
- `fence`：内存屏障
- `fence.i`：指令屏障（Zifencei）

**特权指令**：
- `ecall`：环境调用（系统调用）
- `ebreak`：断点
- `mret`：从机器模式返回

## 小结

本节介绍了 CoralNPU 的指令集概况：

1. **指令集组成**：RV32IMF + Zve32x + Zicsr/Zifencei/Zbb
2. **指令编码**：32 位定长，6 种基本格式（R/I/S/B/U/J）
3. **寄存器**：32 个通用寄存器、32 个浮点寄存器、64 个向量寄存器
4. **指令分类**：标量指令、向量指令、系统指令
