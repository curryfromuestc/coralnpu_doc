# 5.2 RISC-V 基础指令

本章介绍 CoralNPU 支持的 RISC-V 基础指令集及其扩展。CoralNPU 实现了 RV32I 基础整数指令集，并支持 M（乘除法）、F（单精度浮点）、Zicsr（CSR 访问）、Zifencei（指令同步）和 Zbb（位操作）扩展。

## 5.2.1 RV32I 基础整数指令集

RV32I 是 RISC-V 的基础整数指令集，提供了 32 位整数运算、逻辑运算、内存访问和控制流指令。

### 算术运算指令

#### ADD - 加法

**指令格式：** `ADD rd, rs1, rs2`

**功能说明：** 将寄存器 rs1 和 rs2 的值相加，结果写入寄存器 rd。

**解释：** 这里的 `rd`、`rs1`、`rs2` 都是寄存器的名字。RISC-V 有 32 个通用寄存器（x0-x31），其中 x0 永远是 0。这条指令就是把两个寄存器的值加起来，放到第三个寄存器里。

**使用示例：**
```c
// C 代码：c = a + b
// 假设 a 在 x10，b 在 x11，c 在 x12
ADD x12, x10, x11
```

#### ADDI - 立即数加法

**指令格式：** `ADDI rd, rs1, imm`

**功能说明：** 将寄存器 rs1 的值与 12 位立即数 imm 相加，结果写入寄存器 rd。

**解释：** "立即数"（immediate）是指令中直接包含的数值，不需要从寄存器读取。12 位立即数的范围是 -2048 到 2047（有符号数）。

**使用示例：**
```c
// C 代码：c = a + 10
ADDI x12, x10, 10

// 常用技巧：复制寄存器的值
// 相当于 x12 = x10 + 0
ADDI x12, x10, 0
```

#### SUB - 减法

**指令格式：** `SUB rd, rs1, rs2`

**功能说明：** 将寄存器 rs1 的值减去 rs2 的值，结果写入寄存器 rd。

**使用示例：**
```c
// C 代码：c = a - b
SUB x12, x10, x11
```

**注意：** RISC-V 没有 SUBI 指令（立即数减法），如果需要减去一个常数，可以使用 `ADDI rd, rs1, -imm`。

### 逻辑运算指令

#### AND - 按位与

**指令格式：** `AND rd, rs1, rs2`

**功能说明：** 对 rs1 和 rs2 的每一位进行与运算，结果写入 rd。

**解释：** 按位与运算：两个位都是 1 时结果才是 1，否则是 0。例如 1010 & 1100 = 1000。

**使用示例：**
```c
// C 代码：c = a & b
AND x12, x10, x11

// 常用技巧：清除某些位
// 清除最低 4 位：x12 = x10 & 0xFFFFFFF0
ANDI x12, x10, -16
```

#### ANDI - 立即数按位与

**指令格式：** `ANDI rd, rs1, imm`

**功能说明：** 对 rs1 和 12 位立即数进行按位与运算，结果写入 rd。

**使用示例：**
```c
// 提取最低 8 位
ANDI x12, x10, 0xFF
```

#### OR - 按位或

**指令格式：** `OR rd, rs1, rs2`

**功能说明：** 对 rs1 和 rs2 的每一位进行或运算，结果写入 rd。

**解释：** 按位或运算：两个位只要有一个是 1，结果就是 1。例如 1010 | 1100 = 1110。

**使用示例：**
```c
// C 代码：c = a | b
OR x12, x10, x11
```

#### ORI - 立即数按位或

**指令格式：** `ORI rd, rs1, imm`

**功能说明：** 对 rs1 和 12 位立即数进行按位或运算，结果写入 rd。

**使用示例：**
```c
// 设置某些位为 1
ORI x12, x10, 0x0F
```

#### XOR - 按位异或

**指令格式：** `XOR rd, rs1, rs2`

**功能说明：** 对 rs1 和 rs2 的每一位进行异或运算，结果写入 rd。

**解释：** 按位异或运算：两个位不同时结果是 1，相同时是 0。例如 1010 ^ 1100 = 0110。

**使用示例：**
```c
// C 代码：c = a ^ b
XOR x12, x10, x11

// 常用技巧：将寄存器清零
// x12 = x12 ^ x12 = 0
XOR x12, x12, x12
```

#### XORI - 立即数按位异或

**指令格式：** `XORI rd, rs1, imm`

**功能说明：** 对 rs1 和 12 位立即数进行按位异或运算，结果写入 rd。

**使用示例：**
```c
// 翻转某些位
XORI x12, x10, 0xFF
```

### 移位指令

#### SLL - 逻辑左移

**指令格式：** `SLL rd, rs1, rs2`

**功能说明：** 将 rs1 的值左移 rs2 的低 5 位指定的位数，结果写入 rd。

**解释：** 左移就是把二进制数向左移动，右边补 0。例如 0001 左移 2 位变成 0100。只使用 rs2 的低 5 位，所以移位范围是 0-31。

**使用示例：**
```c
// C 代码：c = a << b
SLL x12, x10, x11

// 左移 1 位相当于乘以 2
SLL x12, x10, x11  // 如果 x11 = 1，则 x12 = x10 * 2
```

#### SLLI - 立即数逻辑左移

**指令格式：** `SLLI rd, rs1, shamt`

**功能说明：** 将 rs1 的值左移 shamt 位，结果写入 rd。shamt 是 5 位立即数（0-31）。

**使用示例：**
```c
// 乘以 4（左移 2 位）
SLLI x12, x10, 2
```

#### SRL - 逻辑右移

**指令格式：** `SRL rd, rs1, rs2`

**功能说明：** 将 rs1 的值右移 rs2 的低 5 位指定的位数，左边补 0，结果写入 rd。

**解释：** 逻辑右移：向右移动，左边补 0。例如 1100 右移 2 位变成 0011。

**使用示例：**
```c
// C 代码：c = (unsigned)a >> b
SRL x12, x10, x11
```

#### SRLI - 立即数逻辑右移

**指令格式：** `SRLI rd, rs1, shamt`

**功能说明：** 将 rs1 的值右移 shamt 位，左边补 0，结果写入 rd。

**使用示例：**
```c
// 除以 8（右移 3 位，仅适用于无符号数）
SRLI x12, x10, 3
```

#### SRA - 算术右移

**指令格式：** `SRA rd, rs1, rs2`

**功能说明：** 将 rs1 的值右移 rs2 的低 5 位指定的位数，左边补符号位，结果写入 rd。

**解释：** 算术右移：向右移动，左边补符号位（最高位）。这样可以保持有符号数的符号。例如负数 1100（-4）右移 1 位变成 1110（-2）。

**使用示例：**
```c
// C 代码：c = (signed)a >> b
SRA x12, x10, x11
```

#### SRAI - 立即数算术右移

**指令格式：** `SRAI rd, rs1, shamt`

**功能说明：** 将 rs1 的值右移 shamt 位，左边补符号位，结果写入 rd。

**使用示例：**
```c
// 有符号数除以 4（右移 2 位）
SRAI x12, x10, 2
```

### 比较指令

#### SLT - 有符号比较小于

**指令格式：** `SLT rd, rs1, rs2`

**功能说明：** 如果 rs1 < rs2（有符号比较），则 rd = 1，否则 rd = 0。

**解释：** 这条指令把 rs1 和 rs2 当作有符号数（可以是负数）来比较。

**使用示例：**
```c
// C 代码：c = (a < b) ? 1 : 0
SLT x12, x10, x11

// 判断 a 是否为负数
SLT x12, x10, x0  // 如果 x10 < 0，则 x12 = 1
```

#### SLTI - 立即数有符号比较小于

**指令格式：** `SLTI rd, rs1, imm`

**功能说明：** 如果 rs1 < imm（有符号比较），则 rd = 1，否则 rd = 0。

**使用示例：**
```c
// 判断 a 是否小于 10
SLTI x12, x10, 10
```

#### SLTU - 无符号比较小于

**指令格式：** `SLTU rd, rs1, rs2`

**功能说明：** 如果 rs1 < rs2（无符号比较），则 rd = 1，否则 rd = 0。

**解释：** 这条指令把 rs1 和 rs2 当作无符号数（都是正数）来比较。

**使用示例：**
```c
// C 代码：c = ((unsigned)a < (unsigned)b) ? 1 : 0
SLTU x12, x10, x11

// 常用技巧：判断寄存器是否为 0
// 如果 x10 != 0，则 x12 = 1
SLTU x12, x0, x10
```

#### SLTIU - 立即数无符号比较小于

**指令格式：** `SLTIU rd, rs1, imm`

**功能说明：** 如果 rs1 < imm（无符号比较），则 rd = 1，否则 rd = 0。

**使用示例：**
```c
// 判断无符号数 a 是否小于 100
SLTIU x12, x10, 100
```

### 加载存储指令

#### LW - 加载字

**指令格式：** `LW rd, offset(rs1)`

**功能说明：** 从内存地址 rs1 + offset 处加载 4 字节（32 位）数据到 rd。

**解释：** "字"（word）在 RISC-V 中是 4 字节。offset 是 12 位有符号立即数。这条指令从内存读取数据。

**使用示例：**
```c
// C 代码：c = *a  或  c = a[0]
// 假设 a 的地址在 x10
LW x12, 0(x10)

// C 代码：c = a[2]  （假设 a 是 int 数组）
// 每个 int 是 4 字节，所以偏移是 2*4=8
LW x12, 8(x10)
```

#### LH - 加载半字

**指令格式：** `LH rd, offset(rs1)`

**功能说明：** 从内存地址 rs1 + offset 处加载 2 字节（16 位）数据到 rd，符号扩展到 32 位。

**解释：** "半字"（halfword）是 2 字节。符号扩展是指如果最高位是 1（负数），则高 16 位全部填充 1。

**使用示例：**
```c
// C 代码：c = *(short*)a
LH x12, 0(x10)
```

#### LHU - 加载半字（无符号）

**指令格式：** `LHU rd, offset(rs1)`

**功能说明：** 从内存地址 rs1 + offset 处加载 2 字节数据到 rd，零扩展到 32 位。

**解释：** 零扩展是指高 16 位全部填充 0。

**使用示例：**
```c
// C 代码：c = *(unsigned short*)a
LHU x12, 0(x10)
```

#### LB - 加载字节

**指令格式：** `LB rd, offset(rs1)`

**功能说明：** 从内存地址 rs1 + offset 处加载 1 字节（8 位）数据到 rd，符号扩展到 32 位。

**使用示例：**
```c
// C 代码：c = *(char*)a
LB x12, 0(x10)
```

#### LBU - 加载字节（无符号）

**指令格式：** `LBU rd, offset(rs1)`

**功能说明：** 从内存地址 rs1 + offset 处加载 1 字节数据到 rd，零扩展到 32 位。

**使用示例：**
```c
// C 代码：c = *(unsigned char*)a
LBU x12, 0(x10)
```

#### SW - 存储字

**指令格式：** `SW rs2, offset(rs1)`

**功能说明：** 将 rs2 的低 32 位存储到内存地址 rs1 + offset。

**使用示例：**
```c
// C 代码：*a = b  或  a[0] = b
SW x11, 0(x10)

// C 代码：a[3] = b
SW x11, 12(x10)
```

#### SH - 存储半字

**指令格式：** `SH rs2, offset(rs1)`

**功能说明：** 将 rs2 的低 16 位存储到内存地址 rs1 + offset。

**使用示例：**
```c
// C 代码：*(short*)a = b
SH x11, 0(x10)
```

#### SB - 存储字节

**指令格式：** `SB rs2, offset(rs1)`

**功能说明：** 将 rs2 的低 8 位存储到内存地址 rs1 + offset。

**使用示例：**
```c
// C 代码：*(char*)a = b
SB x11, 0(x10)
```

### 分支跳转指令

#### BEQ - 相等则分支

**指令格式：** `BEQ rs1, rs2, offset`

**功能说明：** 如果 rs1 == rs2，则跳转到 PC + offset，否则继续执行下一条指令。

**解释：** PC（Program Counter）是程序计数器，指向当前指令的地址。offset 是 13 位有符号立即数，必须是 2 的倍数。

**使用示例：**
```c
// C 代码：if (a == b) goto label;
BEQ x10, x11, label

// 判断寄存器是否为 0
BEQ x10, x0, label  // 如果 x10 == 0，跳转
```

#### BNE - 不相等则分支

**指令格式：** `BNE rs1, rs2, offset`

**功能说明：** 如果 rs1 != rs2，则跳转到 PC + offset。

**使用示例：**
```c
// C 代码：if (a != b) goto label;
BNE x10, x11, label

// 判断寄存器是否非 0
BNE x10, x0, label  // 如果 x10 != 0，跳转
```

#### BLT - 有符号小于则分支

**指令格式：** `BLT rs1, rs2, offset`

**功能说明：** 如果 rs1 < rs2（有符号比较），则跳转到 PC + offset。

**使用示例：**
```c
// C 代码：if (a < b) goto label;
BLT x10, x11, label

// 判断是否为负数
BLT x10, x0, label  // 如果 x10 < 0，跳转
```

#### BGE - 有符号大于等于则分支

**指令格式：** `BGE rs1, rs2, offset`

**功能说明：** 如果 rs1 >= rs2（有符号比较），则跳转到 PC + offset。

**使用示例：**
```c
// C 代码：if (a >= b) goto label;
BGE x10, x11, label
```

#### BLTU - 无符号小于则分支

**指令格式：** `BLTU rs1, rs2, offset`

**功能说明：** 如果 rs1 < rs2（无符号比较），则跳转到 PC + offset。

**使用示例：**
```c
// C 代码：if ((unsigned)a < (unsigned)b) goto label;
BLTU x10, x11, label
```

#### BGEU - 无符号大于等于则分支

**指令格式：** `BGEU rs1, rs2, offset`

**功能说明：** 如果 rs1 >= rs2（无符号比较），则跳转到 PC + offset。

**使用示例：**
```c
// C 代码：if ((unsigned)a >= (unsigned)b) goto label;
BGEU x10, x11, label
```

#### JAL - 跳转并链接

**指令格式：** `JAL rd, offset`

**功能说明：** 将下一条指令的地址（PC + 4）保存到 rd，然后跳转到 PC + offset。

**解释：** 这条指令用于函数调用。offset 是 21 位有符号立即数。通常 rd 使用 x1（也叫 ra，return address）。

**使用示例：**
```c
// 调用函数
JAL x1, function  // 保存返回地址到 x1，跳转到 function

// 无条件跳转（不保存返回地址）
JAL x0, label  // x0 永远是 0，所以返回地址被丢弃
```

#### JALR - 跳转并链接寄存器

**指令格式：** `JALR rd, offset(rs1)`

**功能说明：** 将下一条指令的地址（PC + 4）保存到 rd，然后跳转到 rs1 + offset（最低位清零）。

**解释：** 这条指令用于函数返回和间接跳转。最低位清零是为了保证地址对齐。

**使用示例：**
```c
// 函数返回
JALR x0, 0(x1)  // 跳转到 x1 保存的地址

// 间接函数调用
JALR x1, 0(x10)  // 跳转到 x10 指向的地址
```

### 上位立即数指令

#### LUI - 加载上位立即数

**指令格式：** `LUI rd, imm`

**功能说明：** 将 20 位立即数 imm 放到 rd 的高 20 位，低 12 位清零。

**解释：** 这条指令用于加载大的常数。imm 会被放到第 12-31 位，第 0-11 位是 0。

**使用示例：**
```c
// 加载 32 位常数 0x12345678
LUI x10, 0x12345      // x10 = 0x12345000
ADDI x10, x10, 0x678  // x10 = 0x12345678
```

#### AUIPC - PC 相对加载上位立即数

**指令格式：** `AUIPC rd, imm`

**功能说明：** 将 20 位立即数 imm 左移 12 位后与当前 PC 相加，结果写入 rd。

**解释：** 这条指令用于 PC 相对寻址，常用于访问全局变量或跳转到远距离标签。

**使用示例：**
```c
// 获取标签的地址
AUIPC x10, %pcrel_hi(label)  // x10 = PC + (label 的高 20 位偏移)
ADDI x10, x10, %pcrel_lo(label)  // x10 = label 的完整地址
```

### FENCE - 内存屏障

**指令格式：** `FENCE pred, succ`

**功能说明：** 保证 pred 指定的内存操作在 succ 指定的内存操作之前完成。

**解释：** 这条指令用于多核系统中的内存同步。pred 和 succ 可以是 r（读）、w（写）、i（输入）、o（输出）的组合。

**使用示例：**
```c
// 保证所有内存操作完成
FENCE rw, rw
```


## 5.2.2 RV32M 乘除法扩展

RV32M 扩展提供了整数乘法和除法指令。

### 乘法指令

#### MUL - 乘法

**指令格式：** `MUL rd, rs1, rs2`

**功能说明：** 将 rs1 和 rs2 相乘，取结果的低 32 位写入 rd。

**解释：** 两个 32 位数相乘会得到 64 位结果，这条指令只保留低 32 位。

**使用示例：**
```c
// C 代码：c = a * b
MUL x12, x10, x11

// 计算数组索引：index = i * 4
SLLI x12, x10, 2  // 更快的方法
// 或
MUL x12, x10, x13  // x13 = 4
```

#### MULH - 有符号乘法高位

**指令格式：** `MULH rd, rs1, rs2`

**功能说明：** 将 rs1 和 rs2 作为有符号数相乘，取结果的高 32 位写入 rd。

**解释：** 两个有符号数相乘得到 64 位结果，这条指令返回高 32 位（第 32-63 位）。

**使用示例：**
```c
// 获取 64 位乘法结果
MUL x12, x10, x11   // 低 32 位
MULH x13, x10, x11  // 高 32 位
// 现在 x13:x12 包含完整的 64 位结果
```

#### MULHSU - 有符号×无符号乘法高位

**指令格式：** `MULHSU rd, rs1, rs2`

**功能说明：** 将 rs1（有符号）和 rs2（无符号）相乘，取结果的高 32 位写入 rd。

**解释：** 这条指令用于混合符号的乘法，rs1 是有符号数，rs2 是无符号数。

**使用示例：**
```c
// 有符号数乘以无符号数
MULHSU x12, x10, x11  // x10 是有符号，x11 是无符号
```

#### MULHU - 无符号乘法高位

**指令格式：** `MULHU rd, rs1, rs2`

**功能说明：** 将 rs1 和 rs2 作为无符号数相乘，取结果的高 32 位写入 rd。

**使用示例：**
```c
// 无符号 64 位乘法
MUL x12, x10, x11    // 低 32 位
MULHU x13, x10, x11  // 高 32 位
```

### 除法指令

#### DIV - 有符号除法

**指令格式：** `DIV rd, rs1, rs2`

**功能说明：** 将 rs1 除以 rs2（有符号除法），商写入 rd。

**解释：** 这是有符号整数除法，向零舍入。例如 -7 / 2 = -3。

**特殊情况：**
- 除以 0：结果是 -1
- 溢出（-2^31 / -1）：结果是 -2^31

**使用示例：**
```c
// C 代码：c = a / b
DIV x12, x10, x11
```

#### DIVU - 无符号除法

**指令格式：** `DIVU rd, rs1, rs2`

**功能说明：** 将 rs1 除以 rs2（无符号除法），商写入 rd。

**特殊情况：**
- 除以 0：结果是 2^32 - 1（全 1）

**使用示例：**
```c
// C 代码：c = (unsigned)a / (unsigned)b
DIVU x12, x10, x11
```

#### REM - 有符号取余

**指令格式：** `REM rd, rs1, rs2`

**功能说明：** 计算 rs1 除以 rs2 的余数（有符号），结果写入 rd。

**解释：** 余数的符号与被除数（rs1）相同。例如 -7 % 2 = -1。

**特殊情况：**
- 除以 0：结果是 rs1
- 溢出（-2^31 % -1）：结果是 0

**使用示例：**
```c
// C 代码：c = a % b
REM x12, x10, x11

// 判断奇偶性
ANDI x12, x10, 1  // 更快的方法
// 或
REM x12, x10, x13  // x13 = 2
```

#### REMU - 无符号取余

**指令格式：** `REMU rd, rs1, rs2`

**功能说明：** 计算 rs1 除以 rs2 的余数（无符号），结果写入 rd。

**特殊情况：**
- 除以 0：结果是 rs1

**使用示例：**
```c
// C 代码：c = (unsigned)a % (unsigned)b
REMU x12, x10, x11
```

## 5.2.3 RV32F 单精度浮点扩展

RV32F 扩展提供了 32 位（单精度）浮点运算指令。CoralNPU 有 32 个浮点寄存器（f0-f31），每个 32 位。

**解释：** 浮点数是用来表示小数的，遵循 IEEE 754 标准。单精度浮点数有 1 位符号、8 位指数、23 位尾数。

### 浮点算术指令

#### FADD.S - 单精度浮点加法

**指令格式：** `FADD.S fd, fs1, fs2`

**功能说明：** 将浮点寄存器 fs1 和 fs2 的值相加，结果写入 fd。

**解释：** 这里的 fd、fs1、fs2 是浮点寄存器（f0-f31），不是整数寄存器（x0-x31）。

**使用示例：**
```c
// C 代码：float c = a + b
FADD.S f12, f10, f11
```

#### FSUB.S - 单精度浮点减法

**指令格式：** `FSUB.S fd, fs1, fs2`

**功能说明：** 将浮点寄存器 fs1 的值减去 fs2 的值，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = a - b
FSUB.S f12, f10, f11
```

#### FMUL.S - 单精度浮点乘法

**指令格式：** `FMUL.S fd, fs1, fs2`

**功能说明：** 将浮点寄存器 fs1 和 fs2 的值相乘，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = a * b
FMUL.S f12, f10, f11
```

#### FDIV.S - 单精度浮点除法

**指令格式：** `FDIV.S fd, fs1, fs2`

**功能说明：** 将浮点寄存器 fs1 的值除以 fs2 的值，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = a / b
FDIV.S f12, f10, f11
```

#### FSQRT.S - 单精度浮点平方根

**指令格式：** `FSQRT.S fd, fs1`

**功能说明：** 计算 fs1 的平方根，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = sqrt(a)
FSQRT.S f12, f10
```

#### FMADD.S - 单精度浮点乘加

**指令格式：** `FMADD.S fd, fs1, fs2, fs3`

**功能说明：** 计算 (fs1 × fs2) + fs3，结果写入 fd。

**解释：** 这是融合乘加（Fused Multiply-Add，FMA）指令，只进行一次舍入，精度更高，速度更快。

**使用示例：**
```c
// C 代码：float d = a * b + c
FMADD.S f13, f10, f11, f12
```

#### FMSUB.S - 单精度浮点乘减

**指令格式：** `FMSUB.S fd, fs1, fs2, fs3`

**功能说明：** 计算 (fs1 × fs2) - fs3，结果写入 fd。

**使用示例：**
```c
// C 代码：float d = a * b - c
FMSUB.S f13, f10, f11, f12
```

#### FNMADD.S - 单精度浮点负乘加

**指令格式：** `FNMADD.S fd, fs1, fs2, fs3`

**功能说明：** 计算 -(fs1 × fs2) + fs3，结果写入 fd。

**使用示例：**
```c
// C 代码：float d = -(a * b) + c  或  d = c - a * b
FNMADD.S f13, f10, f11, f12
```

#### FNMSUB.S - 单精度浮点负乘减

**指令格式：** `FNMSUB.S fd, fs1, fs2, fs3`

**功能说明：** 计算 -(fs1 × fs2) - fs3，结果写入 fd。

**使用示例：**
```c
// C 代码：float d = -(a * b) - c  或  d = -(a * b + c)
FNMSUB.S f13, f10, f11, f12
```

### 浮点比较指令

#### FEQ.S - 浮点相等比较

**指令格式：** `FEQ.S rd, fs1, fs2`

**功能说明：** 如果 fs1 == fs2，则整数寄存器 rd = 1，否则 rd = 0。

**解释：** 注意结果写入整数寄存器（x0-x31），不是浮点寄存器。

**使用示例：**
```c
// C 代码：int c = (a == b)
FEQ.S x12, f10, f11
```

#### FLT.S - 浮点小于比较

**指令格式：** `FLT.S rd, fs1, fs2`

**功能说明：** 如果 fs1 < fs2，则整数寄存器 rd = 1，否则 rd = 0。

**使用示例：**
```c
// C 代码：int c = (a < b)
FLT.S x12, f10, f11
```

#### FLE.S - 浮点小于等于比较

**指令格式：** `FLE.S rd, fs1, fs2`

**功能说明：** 如果 fs1 <= fs2，则整数寄存器 rd = 1，否则 rd = 0。

**使用示例：**
```c
// C 代码：int c = (a <= b)
FLE.S x12, f10, f11
```

#### FCVT.W.S - 浮点转有符号整数

**指令格式：** `FCVT.W.S rd, fs1`

**功能说明：** 将浮点数 fs1 转换为 32 位有符号整数，结果写入整数寄存器 rd。

**解释：** 转换时会根据舍入模式进行舍入。如果浮点数超出整数范围，结果是最大或最小整数。

**使用示例：**
```c

**指令格式：** `FCVT.WU.S rd, fs1`

**功能说明：** 将浮点数 fs1 转换为 32 位无符号整数，结果写入整数寄存器 rd。

**使用示例：**

// C 代码：unsigned int c = (unsigned int)a
FCVT.WU.S x12, f10
```

#### FCVT.S.W - 有符号整数转浮点

**指令格式：** `FCVT.S.W fd, rs1`

**功能说明：** 将整数寄存器 rs1 的值转换为单精度浮点数，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = (float)a
FCVT.S.W f12, x10
```

#### FCVT.S.WU - 无符号整数转浮点

**指令格式：** `FCVT.S.WU fd, rs1`

**功能说明：** 将整数寄存器 rs1 的值作为无符号数转换为单精度浮点数，结果写入 fd。

**使用示例：**
```c
// C 代码：float c = (float)(unsigned int)a
FCVT.S.WU f12, x10
```

### 浮点符号注入指令

#### FSGNJ.S - 浮点符号注入

**指令格式：** `FSGNJ.S fd, fs1, fs2`

**功能说明：** 将 fs1 的大小与 fs2 的符号组合，结果写入 fd。

**使用示例：**

// 复制浮点数：c = a

#### FSGNJN.S - 浮点符号取反注入

**指令格式：** `FSGNJN.S fd, fs1, fs2`

**功能说明：** 将 fs1 的大小与 fs2 符号的相反值组合，结果写入 fd。

**使用示例：**
```c
// 取负：c = -a
FSGNJN.S f12, f10, f10
```

#### FSGNJX.S - 浮点符号异或注入

**指令格式：** `FSGNJX.S fd, fs1, fs2`

**功能说明：** 将 fs1 的大小与 fs1、fs2 符号的异或值组合，结果写入 fd。

**使用示例：**
```c
// 取绝对值：c = fabs(a)
FSGNJX.S f12, f10, f10
```

### 浮点最值指令

#### FMIN.S - 浮点最小值

**指令格式：** `FMIN.S fd, fs1, fs2`

**功能说明：** 取 fs1 和 fs2 中的较小值，结果写入 fd。

**使用示例：**

### 浮点分类指令

#### FCLASS.S - 浮点分类

**指令格式：** `FCLASS.S rd, fs1`

**解释：** 结果是一个 10 位掩码，每一位表示一种类型：
- 位 0：负无穷
- 位 1：负正常数

// 获取浮点数的位表示

## 5.2.4 Zicsr CSR 指令

Zicsr 扩展提供了访问控制状态寄存器（Control and Status Registers，CSR）的指令。CSR 是特殊的寄存器，用于配置处理器行为、查询状态等。

**解释：** CSR 不同于普通的 x0-x31 寄存器，它们有特殊的功能。每个 CSR 有一个 12 位的地址。常见的 CSR 包括：
- `mstatus`（0x300）：机器状态寄存器
- `mtvec`（0x305）：异常向量基地址
- `mepc`（0x341）：异常 PC
- `mcause`（0x342）：异常原因
- `fcsr`（0x003）：浮点控制状态寄存器
- `frm`（0x002）：浮点舍入模式
- `fflags`（0x001）：浮点异常标志

### CSRRW - CSR 读写

**指令格式：** `CSRRW rd, csr, rs1`

**功能说明：** 将 CSR 的值读到 rd，然后将 rs1 的值写入 CSR。

**解释：** 这是原子操作，先读后写。如果 rd 是 x0，则不读取（可以提高性能）。

**使用示例：**
```c
// 读取并修改 mstatus
CSRRW x10, mstatus, x11  // x10 = 旧的 mstatus，mstatus = x11

// 只写不读
CSRRW x0, mstatus, x11  // mstatus = x11，不读取旧值
```

### CSRRS - CSR 读并置位

**指令格式：** `CSRRS rd, csr, rs1`

**功能说明：** 将 CSR 的值读到 rd，然后将 CSR 与 rs1 按位或，结果写回 CSR。

**解释：** 这条指令用于设置 CSR 中的某些位为 1，而不影响其他位。如果 rs1 是 x0，则只读不写。

**使用示例：**
```c
// 读取 mstatus
CSRRS x10, mstatus, x0  // x10 = mstatus，不修改 mstatus

// 设置 mstatus 的某些位
CSRRS x10, mstatus, x11  // x10 = 旧的 mstatus，mstatus |= x11
```

### CSRRC - CSR 读并清除位

**指令格式：** `CSRRC rd, csr, rs1`

**功能说明：** 将 CSR 的值读到 rd，然后将 CSR 与 ~rs1 按位与，结果写回 CSR。

**解释：** 这条指令用于清除 CSR 中的某些位为 0，而不影响其他位。如果 rs1 是 x0，则只读不写。

**使用示例：**
```c
// 清除 mstatus 的某些位
CSRRC x10, mstatus, x11  // x10 = 旧的 mstatus，mstatus &= ~x11
```

### CSRRWI - CSR 读写立即数

**指令格式：** `CSRRWI rd, csr, imm`

**功能说明：** 将 CSR 的值读到 rd，然后将 5 位立即数 imm（零扩展到 32 位）写入 CSR。

**解释：** 这是 CSRRW 的立即数版本，imm 是 5 位无符号数（0-31）。

**使用示例：**
```c
// 将 CSR 设置为小常数
CSRRWI x10, fcsr, 0  // 清除浮点状态，x10 = 旧的 fcsr
```

### CSRRSI - CSR 读并置位立即数

**指令格式：** `CSRRSI rd, csr, imm`

**功能说明：** 将 CSR 的值读到 rd，然后将 CSR 与 5 位立即数 imm 按位或，结果写回 CSR。

**使用示例：**
```c
// 设置 CSR 的某些位
CSRRSI x10, mstatus, 0x8  // 设置位 3
```

### CSRRCI - CSR 读并清除位立即数

**指令格式：** `CSRRCI rd, csr, imm`

**功能说明：** 将 CSR 的值读到 rd，然后将 CSR 与 ~imm 按位与，结果写回 CSR。

**使用示例：**
```c
// 清除 CSR 的某些位
CSRRCI x10, mstatus, 0x8  // 清除位 3
```

### 常用 CSR 操作示例

```c
// 读取浮点舍入模式
CSRRS x10, frm, x0

// 设置浮点舍入模式为向零舍入（RZ = 1）
CSRRWI x0, frm, 1

// 清除浮点异常标志
CSRRWI x0, fflags, 0

// 读取并清除浮点异常标志
CSRRW x10, fflags, x0
```

## 5.2.5 Zifencei 指令

Zifencei 扩展提供了指令同步指令，用于确保指令缓存与内存一致。

### FENCE.I - 指令同步

**指令格式：** `FENCE.I`

**功能说明：** 刷新指令缓存，确保后续取指令操作能看到之前的内存写入。

**解释：** 当程序修改了代码段（例如动态生成代码、加载新代码），需要使用这条指令来确保处理器执行的是新代码，而不是缓存中的旧代码。

**使用示例：**
```c
// 动态代码生成后
SW x10, 0(x11)    // 写入新指令到内存
SW x12, 4(x11)    // 写入更多指令
FENCE.I           // 刷新指令缓存
JALR x0, 0(x11)   // 跳转到新代码
```

**应用场景：**
- JIT（即时编译）系统
- 动态链接器
- 自修改代码
- 代码热更新

## 5.2.6 Zbb 位操作扩展

Zbb（Basic Bit-manipulation）扩展提供了常用的位操作指令，可以提高某些算法的效率。

### 位逻辑指令

#### ANDN - 与非

**指令格式：** `ANDN rd, rs1, rs2`

**功能说明：** 计算 rs1 & ~rs2，结果写入 rd。

**解释：** 这条指令用于清除 rs1 中某些位，这些位由 rs2 指定。

**使用示例：**
```c
// C 代码：c = a & ~b
ANDN x12, x10, x11

// 清除 rs1 的最低 8 位
ADDI x13, x0, 0xFF
ANDN x12, x10, x13
```

#### ORN - 或非

**指令格式：** `ORN rd, rs1, rs2`

**功能说明：** 计算 rs1 | ~rs2，结果写入 rd。

**使用示例：**
```c
// C 代码：c = a | ~b
ORN x12, x10, x11
```

#### XNOR - 异或非

**指令格式：** `XNOR rd, rs1, rs2`

**功能说明：** 计算 ~(rs1 ^ rs2)，结果写入 rd。

**解释：** 异或非也叫同或，当两个位相同时结果是 1。

**使用示例：**
```c
// C 代码：c = ~(a ^ b)
XNOR x12, x10, x11

// 判断两个数是否相等（所有位都相同）
XNOR x12, x10, x11
ADDI x13, x0, -1   // x13 = 0xFFFFFFFF
BEQ x12, x13, equal
```

### 位计数指令

#### CLZ - 前导零计数

**指令格式：** `CLZ rd, rs1`

**功能说明：** 计算 rs1 从最高位开始有多少个连续的 0，结果写入 rd。

**解释：** 例如 0x00FF0000 有 8 个前导零，0x80000000 有 0 个前导零，0x00000000 有 32 个前导零。

**使用示例：**
```c
// C 代码：c = __builtin_clz(a)
CLZ x12, x10

// 计算整数的位宽（最高位的位置）
CLZ x12, x10
ADDI x13, x0, 32
SUB x12, x13, x12  // x12 = 32 - clz(x10)
```

#### CTZ - 尾随零计数

**指令格式：** `CTZ rd, rs1`

**功能说明：** 计算 rs1 从最低位开始有多少个连续的 0，结果写入 rd。

**解释：** 例如 0x00FF0000 有 16 个尾随零，0x00000001 有 0 个尾随零，0x00000000 有 32 个尾随零。

**使用示例：**
```c
// C 代码：c = __builtin_ctz(a)
CTZ x12, x10

// 找到最低位的 1 的位置
CTZ x12, x10
```

#### CPOP - 置位计数

**指令格式：** `CPOP rd, rs1`

**功能说明：** 计算 rs1 中有多少个 1，结果写入 rd。

**解释：** 也叫人口计数（population count）或汉明重量（Hamming weight）。

**使用示例：**
```c
// C 代码：c = __builtin_popcount(a)
CPOP x12, x10

// 判断是否为 2 的幂（只有一个 1）
CPOP x12, x10
ADDI x13, x0, 1
BEQ x12, x13, is_power_of_2
```

#### MIN - 有符号最小值

**指令格式：** `MIN rd, rs1, rs2`

### 符号扩展指令

#### SEXT.B - 字节符号扩展

**指令格式：** `SEXT.B rd, rs1`

**功能说明：** 将 rs1 的低 8 位符号扩展到 32 位，结果写入 rd。

**解释：** 如果第 7 位是 1（负数），则高 24 位填充 1；否则填充 0。

**使用示例：**
```c
// C 代码：c = (int)(char)a
SEXT.B x12, x10

// 等价于（但更快）：
SLLI x12, x10, 24
SRAI x12, x12, 24
```

#### SEXT.H - 半字符号扩展

**指令格式：** `SEXT.H rd, rs1`

**功能说明：** 将 rs1 的低 16 位符号扩展到 32 位，结果写入 rd。

**使用示例：**
```c
// C 代码：c = (int)(short)a
SEXT.H x12, x10

// 等价于（但更快）：
SLLI x12, x10, 16
SRAI x12, x12, 16
```

#### ZEXT.H - 半字零扩展

**指令格式：** `ZEXT.H rd, rs1`

**功能说明：** 将 rs1 的低 16 位零扩展到 32 位，结果写入 rd。

**解释：** 高 16 位填充 0。

**使用示例：**
```c
// C 代码：c = (unsigned short)a
ZEXT.H x12, x10

// 等价于（但可能更快）：
LUI x13, 0x10
ADDI x13, x13, -1  // x13 = 0xFFFF
AND x12, x10, x13
```

### 循环移位指令

#### ROL - 循环左移

**指令格式：** `ROL rd, rs1, rs2`

**功能说明：** 将 rs1 循环左移 rs2 的低 5 位指定的位数，结果写入 rd。

**解释：** 循环移位是指移出的位从另一端移入。例如 0x12345678 循环左移 4 位变成 0x23456781。

**使用示例：**
```c
// C 代码：c = (a << n) | (a >> (32 - n))
ROL x12, x10, x11
```

#### ROR - 循环右移

**指令格式：** `ROR rd, rs1, rs2`

**功能说明：** 将 rs1 循环右移 rs2 的低 5 位指定的位数，结果写入 rd。

**使用示例：**
```c
// C 代码：c = (a >> n) | (a << (32 - n))
ROR x12, x10, x11
```

#### RORI - 立即数循环右移

**指令格式：** `RORI rd, rs1, shamt`

**功能说明：** 将 rs1 循环右移 shamt 位，结果写入 rd。

**使用示例：**
```c
// 循环右移 8 位
RORI x12, x10, 8
```

### 字节操作指令

#### ORCB - 字节或合并

**指令格式：** `ORCB rd, rs1`

**功能说明：** 对 rs1 的每个字节，如果该字节非零，则结果对应字节为 0xFF，否则为 0x00。

**解释：** 这条指令可以快速检测哪些字节非零。

**使用示例：**
```c
// 检测字符串结束符（零字节）
ORCB x12, x10
// 如果 x12 != 0xFFFFFFFF，说明有零字节
```

#### REV8 - 字节反转

**指令格式：** `REV8 rd, rs1`

**功能说明：** 反转 rs1 的字节顺序，结果写入 rd。

**解释：** 这条指令用于大小端转换。例如 0x12345678 变成 0x78563412。

**使用示例：**
```c
// C 代码：c = __builtin_bswap32(a)
REV8 x12, x10

// 大小端转换
LW x10, 0(x11)    // 从内存加载
REV8 x10, x10     // 转换字节序
```

## 5.2.7 系统指令

除了上述指令，CoralNPU 还支持一些系统控制指令。

### ECALL - 环境调用

**指令格式：** `ECALL`

**功能说明：** 触发环境调用异常，用于系统调用。

**解释：** 这条指令用于从用户态切换到内核态，或者在裸机环境中调用监控程序。

**使用示例：**
```c
// 系统调用
ADDI x17, x0, 93  // 系统调用号（exit）
ADDI x10, x0, 0   // 参数（退出码）
ECALL
```

### EBREAK - 断点

**指令格式：** `EBREAK`

**功能说明：** 触发断点异常，用于调试。

**解释：** 调试器可以在这条指令处暂停程序执行。

**使用示例：**
```c
// 调试断点
EBREAK
```

### MRET - 机器模式返回

**指令格式：** `MRET`

**功能说明：** 从机器模式异常处理程序返回。

**解释：** 这条指令恢复 PC 和状态，返回到异常发生前的位置。

**使用示例：**
```c
// 异常处理程序结尾
MRET
```

### WFI - 等待中断

**指令格式：** `WFI`

**功能说明：** 等待中断，处理器进入低功耗状态。

**解释：** 这条指令让处理器休眠，直到有中断发生。

**使用示例：**
```c
// 空闲循环
loop:
    WFI
    J loop
```

## 5.2.8 伪指令

RISC-V 汇编器支持许多伪指令（pseudo-instructions），它们会被翻译成一条或多条真实指令。以下是常用的伪指令：

### 数据移动

```c
NOP                 // 空操作 → ADDI x0, x0, 0
MV rd, rs           // 移动 → ADDI rd, rs, 0
LI rd, imm          // 加载立即数 → LUI + ADDI（如果需要）
LA rd, symbol       // 加载地址 → AUIPC + ADDI
```

### 跳转

```c
J offset            // 跳转 → JAL x0, offset
JR rs               // 寄存器跳转 → JALR x0, 0(rs)
RET                 // 返回 → JALR x0, 0(x1)
CALL offset         // 调用 → AUIPC + JALR
TAIL offset         // 尾调用 → AUIPC + JALR
```

### 比较分支

```c
BEQZ rs, offset     // 等于零则分支 → BEQ rs, x0, offset
BNEZ rs, offset     // 非零则分支 → BNE rs, x0, offset
BLEZ rs, offset     // 小于等于零则分支 → BGE x0, rs, offset
BGEZ rs, offset     // 大于等于零则分支 → BGE rs, x0, offset
BLTZ rs, offset     // 小于零则分支 → BLT rs, x0, offset
BGTZ rs, offset     // 大于零则分支 → BLT x0, rs, offset
BGT rs, rt, offset  // 大于则分支 → BLT rt, rs, offset
BLE rs, rt, offset  // 小于等于则分支 → BGE rt, rs, offset
BGTU rs, rt, offset // 无符号大于则分支 → BLTU rt, rs, offset
BLEU rs, rt, offset // 无符号小于等于则分支 → BGEU rt, rs, offset
```

### 取反

```c
NOT rd, rs          // 按位取反 → XORI rd, rs, -1
NEG rd, rs          // 取负 → SUB rd, x0, rs
NEGW rd, rs         // 字取负 → SUBW rd, x0, rs（RV64）
```

### 比较

```c
SEQZ rd, rs         // 等于零则置位 → SLTIU rd, rs, 1
SNEZ rd, rs         // 非零则置位 → SLTU rd, x0, rs
SLTZ rd, rs         // 小于零则置位 → SLT rd, rs, x0
SGTZ rd, rs         // 大于零则置位 → SLT rd, x0, rs
```

### CSR 操作

```c
CSRR rd, csr        // 读 CSR → CSRRS rd, csr, x0
CSRW csr, rs        // 写 CSR → CSRRW x0, csr, rs
CSRS csr, rs        // 置位 CSR → CSRRS x0, csr, rs
CSRC csr, rs        // 清除 CSR → CSRRC x0, csr, rs
CSRWI csr, imm      // 写 CSR 立即数 → CSRRWI x0, csr, imm
CSRSI csr, imm      // 置位 CSR 立即数 → CSRRSI x0, csr, imm
CSRCI csr, imm      // 清除 CSR 立即数 → CSRRCI x0, csr, imm
```

### 浮点

```c
FMOV.S fd, fs       // 浮点移动 → FSGNJ.S fd, fs, fs
FABS.S fd, fs       // 浮点绝对值 → FSGNJX.S fd, fs, fs
FNEG.S fd, fs       // 浮点取负 → FSGNJN.S fd, fs, fs
```

## 5.2.9 寄存器约定

RISC-V 定义了标准的寄存器使用约定（ABI），以便不同编译器生成的代码可以互相调用。

### 整数寄存器

| 寄存器 | ABI 名称 | 说明 | 调用者/被调用者保存 |
|--------|----------|------|---------------------|
| x0 | zero | 硬件零 | - |
| x1 | ra | 返回地址 | 调用者 |
| x2 | sp | 栈指针 | 被调用者 |
| x3 | gp | 全局指针 | - |
| x4 | tp | 线程指针 | - |
| x5-x7 | t0-t2 | 临时寄存器 | 调用者 |
| x8 | s0/fp | 保存寄存器/帧指针 | 被调用者 |
| x9 | s1 | 保存寄存器 | 被调用者 |
| x10-x11 | a0-a1 | 参数/返回值 | 调用者 |
| x12-x17 | a2-a7 | 参数 | 调用者 |
| x18-x27 | s2-s11 | 保存寄存器 | 被调用者 |
| x28-x31 | t3-t6 | 临时寄存器 | 调用者 |

**解释：**
- **调用者保存**：函数调用前，调用者需要保存这些寄存器的值（如果需要的话）
- **被调用者保存**：被调用的函数需要保存并恢复这些寄存器的值
- **参数传递**：前 8 个参数通过 a0-a7 传递，更多参数通过栈传递
- **返回值**：通过 a0-a1 返回（a0 是主返回值，a1 用于 64 位返回值的高 32 位）

### 浮点寄存器

| 寄存器 | ABI 名称 | 说明 | 调用者/被调用者保存 |
|--------|----------|------|---------------------|
| f0-f7 | ft0-ft7 | 临时寄存器 | 调用者 |
| f8-f9 | fs0-fs1 | 保存寄存器 | 被调用者 |
| f10-f11 | fa0-fa1 | 参数/返回值 | 调用者 |
| f12-f17 | fa2-fa7 | 参数 | 调用者 |
| f18-f27 | fs2-fs11 | 保存寄存器 | 被调用者 |
| f28-f31 | ft8-ft11 | 临时寄存器 | 调用者 |

### 函数调用示例

```c
// C 代码
int add(int a, int b) {
    return a + b;
}

int main() {
    int c = add(3, 5);
    return c;
}

// 汇编代码
add:
    ADD a0, a0, a1    // a0 = a0 + a1
    RET               // 返回

main:
    ADDI sp, sp, -16  // 分配栈空间
    SW ra, 12(sp)     // 保存返回地址
    
    ADDI a0, x0, 3    // 第一个参数
    ADDI a1, x0, 5    // 第二个参数
    CALL add          // 调用函数
    
    LW ra, 12(sp)     // 恢复返回地址
    ADDI sp, sp, 16   // 释放栈空间
    RET               // 返回
```

## 5.2.10 总结

本章介绍了 CoralNPU 支持的 RISC-V 指令集：

1. **RV32I**：基础整数指令，包括算术、逻辑、移位、比较、加载存储、分支跳转等
2. **RV32M**：乘除法扩展，提供高效的整数乘除法运算
3. **RV32F**：单精度浮点扩展，支持 IEEE 754 浮点运算
4. **Zicsr**：CSR 访问指令，用于系统配置和状态查询
5. **Zifencei**：指令同步，用于动态代码生成
6. **Zbb**：位操作扩展，提供高效的位运算指令

这些指令集为 CoralNPU 提供了完整的标量计算能力，配合向量和矩阵指令，可以高效地执行机器学习和信号处理任务。
