# 项目介绍

## 项目概述

CoralNPU 是由 Google Research 设计并开源的机器学习推理硬件加速器。它是一个神经处理单元（NPU），也被称为 AI 加速器或深度学习处理器。CoralNPU 基于 32 位 RISC-V 指令集架构（ISA），专门针对超低功耗的可穿戴设备（如耳戴式设备、增强现实眼镜和智能手表）中的机器学习推理任务进行了优化。

CoralNPU 是一个完全开源的 IP 核心，可以免费集成到片上系统（SoC）中。它的设计理念是从机器学习加速器的数据平面特性出发，构建了一个带有自定义 SIMD 指令的 RISC-V CPU，并在微架构层面做出了与 ML 加速器特性相匹配的决策。

### 设计出发点

CoralNPU 的设计从矩阵（domain/matrix）能力开始，然后添加向量（vector）和标量（scalar）能力，形成一个融合的设计。这种设计方法与传统的 CPU 设计不同——传统 CPU 通常从标量核心开始，然后添加向量扩展；而 CoralNPU 则是以 ML 计算的核心需求（矩阵运算）为起点，反向构建整个处理器架构。

## 核心特性

CoralNPU 提供了以下主要特性：

### 指令集架构



- **RISC-V ISA**：支持 RV32IMF\_Zve32x 指令集（具体为 `rv32imf\_zve32x\_zicsr\_zifencei\_zbb`）
- **32 位地址空间**：支持应用程序和操作系统内核
- **自定义 SIMD 指令**：回收了 C 扩展编码空间（按照 RISC-V 规范允许），用于提供 SIMD 寄存器的 6 位索引和灵活的类型编码



### 流水线架构



- **四级流水线**：指令获取、解码/分发、执行/写回
- **顺序分发，乱序退休**：保证指令按顺序进入流水线，但可以乱序完成
- **多发射能力**：
  

  - 标量指令：每周期最多 4 条
  - 向量指令：每周期最多 2 条
  

- **无推测执行**：简化设计，降低功耗



### 内存系统



- **ITCM（指令紧耦合内存）**：8 KB，单周期延迟 SRAM
- **DTCM（数据紧耦合内存）**：32 KB，单周期延迟 SRAM
- **L1 指令缓存**：8 KB，4 路组相联，256 位块大小
- **L1 数据缓存**：16 KB，4 路组相联，双 bank 架构（每个 bank 8 KB）
- 紧耦合内存比缓存更高效，提供可预测的低延迟访问



### SIMD/向量处理



- **SIMD 宽度**：128 位（当前），256 位（未来）
- **向量寄存器**：64 个，每个 256 位（例如 8 个 int32）
- **累加器**：8x8 阵列，每个 32 位
- **支持的数据宽度**：8 位、16 位、32 位
- **条带挖掘（Stripmining）**：单条前端指令可以展开为 4 条串行执行的 SIMD 指令，减少指令分发压力



### 矩阵运算引擎



- **外积 MAC（乘累加）引擎**：量化的外积乘累加引擎
- **计算能力**：每周期 256 次 MAC 操作
- **架构**：4x 8 位乘法器，结果累加到 32 位累加器
- **二维广播结构**：最大化计算与内存访问的比率



### 总线接口



- **AXI4 总线接口**：可作为管理者（manager）和从属（subordinate）
- **外部内存访问**：与外部内存交互
- **外部配置**：允许外部 CPU 配置 CoralNPU

### 标量核心（Scalar Core）

标量核心是一个简单的 RISC-V 前端，负责驱动 ML+SIMD 后端的命令队列。

**主要特点：**



- **指令集**：RV32IM（32 位整数 + 乘除法）
- **执行模型**：运行到完成（run-to-completion）模型，无操作系统，无中断
- **流水线**：顺序执行，无推测
- **分支预测**：简单的静态预测策略
  

  - 向后分支（backward branches）：预测为跳转
  - 向前分支（forward branches）：预测为不跳转
  - 预测错误时产生一个周期的惩罚
  

- **寄存器**：31 个通用寄存器（x1-x31，x0 固定为 0）+ 控制状态寄存器（CSR）



标量核心的设计目标是尽可能小和简单，因为它只是整个计算流水线的"开销"部分。所有的控制任务都被卸载到系统管理控制器（SMC）。

### 向量核心（Vector Core）

向量核心提供 SIMD（单指令多数据）处理能力。在 CoralNPU 中，"SIMD"和"向量"这两个术语可以互换使用，指的是一个简单实用的 SIMD 指令定义，没有可变长度行为。

**主要特点：**



- **解耦设计**：标量前端通过 FIFO 结构与后端解耦，缓冲向量指令
- **依赖解析**：只有当向量寄存器文件中的依赖关系解决后，指令才会发送到相关的命令队列
- **寄存器**：
  

  - 64 个向量寄存器（v0-v63），每个 256 位
  - 8x8 累加器阵列，每个累加器 32 位
  

- **数据宽度**：支持 8 位、16 位、32 位数据类型



**条带挖掘（Stripmining）机制：**

条带挖掘是将基于数组的并行性折叠以适应可用硬件并行性的技术。为了减少前端指令分发压力成为瓶颈，并原生支持通过 SIMD 寄存器的指令级平铺模式，指令编码明确包含了条带挖掘机制。

例如：


- 前端分发一条 `vadd v0` 指令
- 在发射阶段展开为 4 条串行指令：`vadd v0 : vadd v1 : vadd v2 : vadd v3`
- 这 4 条指令作为离散事件依次处理



这种机制大大减少了指令分发的压力，同时保持了高计算吞吐量。

### 矩阵运算引擎（MAC Engine）

矩阵运算引擎是 CoralNPU 设计的核心组件，提供量化的外积乘累加能力。

**外积架构：**

外积引擎提供二维广播结构，以最大化可交付计算量与内存访问的比率：


- **一个轴**：并行广播（"宽"，卷积权重）
- **另一个轴**：多个批次的转置移位输入（"窄"，例如 MobileNet XY 批处理）



**计算能力：**


- **MAC 单元**：4x 8 位乘法器
- **累加器**：32 位
- **吞吐量**：每周期 256 次 MAC 操作



**VDOT 指令：**

外积结构是多个 VDOT 操作码的垂直排列。每个 VDOT 使用 4 个 8 位乘法，结果归约到 32 位累加器中。

这种架构特别适合卷积神经网络（CNN）中的卷积操作，可以高效地处理量化的神经网络模型。

## 设计理念

### 为什么这样设计？

CoralNPU 的设计理念源于对机器学习推理工作负载的深入理解：

**1. 计算密集型优先**

机器学习推理的核心是大量的矩阵乘法和卷积操作。CoralNPU 从这个核心需求出发，首先设计了高效的矩阵运算引擎（MAC），然后围绕它构建其他组件。这与传统 CPU 设计（先有标量核心，再添加向量扩展）的方向相反。

**2. 量化支持**

现代移动和嵌入式 ML 模型广泛使用量化技术（通常是 8 位整数）来减少模型大小和计算量。CoralNPU 的 MAC 引擎原生支持 8 位量化计算，并提供 32 位累加器以避免溢出。

**3. 外积架构**

外积（outer product）架构通过二维广播结构，可以用最少的内存访问实现最多的计算。这对于内存带宽受限的嵌入式系统至关重要。

**4. 解耦的前后端**

标量前端和向量/矩阵后端通过 FIFO 解耦，允许：


- 前端以自己的节奏分发指令
- 后端根据数据依赖关系调度执行
- 隐藏内存延迟和计算延迟



**5. 紧耦合内存**

使用 ITCM 和 DTCM 而不是纯缓存架构，提供：


- 可预测的单周期延迟
- 更低的功耗（相比缓存）
- 更简单的软件模型



**6. 条带挖掘**

通过指令编码中的条带挖掘机制，减少了指令分发压力，同时原生支持循环展开和平铺优化。

**7. 简化的标量核心**

标量核心被设计得尽可能简单：


- 无操作系统支持
- 无中断
- 无推测执行
- 简单的分支预测



这是因为在 ML 推理场景中，大部分时间都花在向量/矩阵计算上，标量核心只需要做好循环控制和地址生成即可。

### 面向的应用场景

CoralNPU 专门针对以下应用场景进行了优化：

**1. 超低功耗可穿戴设备**


- 耳戴式设备（hearables）：语音识别、降噪、语音增强
- AR 眼镜：物体识别、场景理解、手势识别
- 智能手表：健康监测、活动识别、语音助手



**2. 边缘 AI 推理**


- 实时推理：低延迟要求
- 离线运行：无需云端连接
- 隐私保护：数据不离开设备



**3. 量化神经网络**


- 8 位整数量化模型
- MobileNet、EfficientNet 等轻量级模型
- 卷积神经网络（CNN）



**4. 批处理推理**


- 支持小批量并行处理
- 提高硬件利用率
- 适合实时流式数据处理



**不适合的场景：**


- 训练任务（CoralNPU 专注于推理）
- 高精度浮点计算（虽然支持 F 扩展，但主要优化是针对量化整数）
- 通用计算任务（这不是一个通用 CPU）

**注意事项：**



- **SIMD vs 向量**：在 CoralNPU 文档中，这两个术语通常可以互换使用。严格来说，CoralNPU 的向量指令是固定长度的 SIMD 指令，不是可变长度的向量指令（如 ARM SVE 或 RISC-V RVV）。

- **C 扩展的回收**：CoralNPU 回收了 RISC-V C 扩展的编码空间用于自定义 SIMD 指令。这意味着 CoralNPU 不支持标准的 RISC-V 压缩指令，但获得了更大的编码空间来支持 64 个向量寄存器（需要 6 位索引）。

- **运行到完成模型**：CoralNPU 采用的执行模型不支持操作系统和中断。这简化了硬件设计，但也意味着它主要用于专用的 ML 推理任务，而不是通用计算。

- **量化优先**：虽然 CoralNPU 支持浮点运算（F 扩展），但其硬件优化主要针对 8 位整数量化计算。如果你的应用需要高精度浮点计算，CoralNPU 可能不是最佳选择。
